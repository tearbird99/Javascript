### 🔖 개요
객체를 확장하거나 복사할 때 가장 중요하게 다뤄지는 개념이 바로 **얕은 복사(Shallow Copy)**와 **깊은 복사(Deep Copy)**이다. 자바스크립트에서 객체는 참조 타입이기 때문에 값이 아닌 메모리 주소가 복사된다. 이 때문에 복사를 제대로 하지 않으면 원본 객체까지 변경되는 버그가 발생할 수 있다.

---

### ⭐ 얕은 복사 (Shallow Copy)
얕은 복사는 객체의 최상위 속성만 복사하는 방식이다. 객체 내부의 프로퍼티 중 참조 타입인 객체, 배열 등은 메모리 주소값만 복사된다. 객체의 구조가 평면적일 때는 문제가 없지만, 객체 안에 또 다른 객체나 배열이 들어있는 구조에서는 그 내부까지 복사되지 않고 참조만 이어지게 된다.
- **특징**: 복사본 내부의 중첩된 객체를 수정하면 원본의 중첩된 객체도 같이 바뀐다.
- **방법**: Object.assign(), 스프레드 연산자(...) 등


#### 🌟 spread 연산자
```javascript
const user = { name: "Edery", address: { city: "서울" } };
const copy = { ...user };

copy.name = "홍길동";
copy.address.city = "부산";

console.log(user.name); // Edery
console.log(user.address.city); // 부산
```

#### 🌟 Object.assign()
```javascript
const copy2 = Object.assign({}, user);
```

---

### ⭐ 깊은 복사 (Deep Copy)
얕은 복사와 달리, 객체 내부에 또 다른 객체나 배열이 있을 경우에도 재귀적으로 복사해 준다. 복사본의 모든 프로퍼티는 원본과 전혀 다른 메모리 주소를 가지므로, 어떤 값을 수정하더라도 서로 영향을 주지 않는다.

#### 🌟 JSON 방식 (직렬화+객체화)
JSON.stringify()로 직렬화(객체->문자열) 후 JSON.parse()로 객체화(문자열->객체)하는 방식이다. 하지만 JSON 문법으로 자바스크립트의 값 전체를 표현하지 못하므로 한계점이 있다.
```javascript
const deepCopy = JSON.parse(JSON.stringify(user));
```

#### 🌟 structuredClone()
structuredClone()은 최신 브라우저(크롬, 파이어폭스, 엣지)에서 지원하는 내장 함수로, 거의 모든 객체 타입을 깊은 복사할 수 있다. 순환 참조도 문제없이 처리하고, 대부분의 내장 객체(Date, RegExp, Blob)들도 안정적으로 복제한다.
```javascript
const deepCopy2 = structuredClone(user);
```

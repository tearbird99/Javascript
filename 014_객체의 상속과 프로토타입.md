### 🔖 개요
자바스크립트의 객체에는 자체 프로퍼티와 프로토타입으로부터 상속받은 프로퍼티가 있다. 객체의 프로토타입(prototype) 속성은 자신이 어디에서 프로퍼티를 상속했는지를 나타내는 체인(chain)을 형성하는데, 이를 **프로토타입 체인**이라고 한다.

---

### ⭐ 프로토타입 체인
1. 객체 obj에 x 프로퍼티를 가져온다고 가정해보자.
2. obj에 x라는 자체 프로퍼티가 없다면 obj의 프로토타입 객체에서 x 프로퍼티를 검색한다.
3. 프로토타입 객체에도 자체 x 프로퍼티가 없지만 그 객체의 프로토타입이 있다면, 프로토타입의 프로토타입에서 x 프로퍼티를 검색한다.

즉, x 프로퍼티를 찾거나 프로토타입이 null인 객체에 도달할 때까지 검색을 계속하게 된다. 이렇게 형성되는 것이 프로토타입 체인이다.
```javascript
let obj = {}; // Object.prototype에서 객체 메서드를 상속
obj.x = 10; // 자체 프로퍼티 x 생성

let prop = Object.create(obj); // prop은 obj와 Object.prototype에서 프로퍼티를 상속
prop.y = 20; // 자체 프로퍼티 y 생성

let props = Object.create(prop); // props는 obj, prop, Object.prototype에서 프로퍼티를 상속
props.z = 30; // 프로퍼티 z 생성

console.log(props); // { z: 30 }
console.log(props.x + props.y); // 10+20=30
```
만약 프로토타입의 프로퍼티와 자체 생성한 프로퍼티가 겹친다면, 상속된 프로퍼티는 새로운 프로퍼티에 가려지게 된다.

---

### ⭐ 프로퍼티 접근 에러
존재하지 않는 프로퍼티를 검색 ➡️ undefined
존재하지 않는 '객체'의 프로퍼티를 검색 ➡️ 에러
```
let subtitle = book.subtitle; // undefined
let len = book.subtitle.length; // TypeError: undefined는 객체가 아니므로 점근하면 에러 (null도 마찬가지)
```

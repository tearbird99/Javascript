### 🔖 개요

자바스크립트에서 배열은 내부적으로 **숫자 형태의 문자열을 키(Key)로 가지고, length라는 특별한 속성을 관리하는 특수한 객체**로 구현되어 있다.

---

### ⭐ 배열 $\subset$ 객체
typeof로 배열의 타입을 출력해 보면 object가 나온다.
```javascript
const arr = [1, 2, 3];
console.log(typeof arr); // object 출력
```

---

### ⭐ 배열과 일반 객체의 구조 비교
배열은 내부적으로 일반 객체와 매우 유사하게 작동한다.

| 구분 | 일반 객체 ( {} ) | 배열 ( [] ) |
| :--- | :---: | :---: |
| **데이터 접근** | `obj.key` / `obj['key']` | `arr[index]` (숫자 인덱스) |
| **핵심 목적** | 이름(키) 기반 데이터 구조화 | 순서 기반 목록 관리 |
| **주요 속성** | 사용자 정의 프로퍼티 | `length` (자동 갱신) |
| **프로토타입** | `Object.prototype` 상속 | `Array.prototype` 상속 |
| **데이터 순서** | 보장하지 않음 (순서 없음) | 항상 보장함 (순서 있음) |

데이터 접근 시 배열에서 숫자를 사용하는 것은 자바스크립트 엔진이 내부적으로 문자열로 변환하여 처리하는 것이므로, **객체의 키가 문자열이라는 원칙은 동일하게 유지**된다.

---

### ⭐ 프로토타입의 차이
배열이 일반 객체와 결정적으로 다른 점은 **상속받는 부모(프로토타입)**에 있다.

- **일반 객체**: **Object.prototype을 상속**받는다. (toString(), hasOwnProperty() 등 사용)
- **배열**: **Array.prototype을 상속**받는다. (push(), pop(), map(), filter() 등 강력한 배열 전용 메서드 사용)
